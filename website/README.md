# llm-d website

The [llm-d.ai](https://llm-d.ai) website — **landing page, documentation, blog, and
community** — built with [Docusaurus](https://docusaurus.io) and living **inside the
`llm-d` repo** (`website/`), co-located with the docs it publishes.

This is an **experimental clean rebuild** of the current site (today built from the
separate `llm-d.github.io` repo via a multi-app `preview/` + remote-content + release-branch
pipeline). The goal: one self-contained, easy-to-contribute folder following Docusaurus
conventions.

## Quick start

```bash
cd website
npm install
npm start        # dev server at http://localhost:3000
npm run build    # production build -> ./build
npm run serve    # serve the production build
```

## Layout

```
website/
├── docs/                 # The docs (committed). Ordering/labels/collapse via _category_.json.
│   └── how-to-guides/    #   Deployment guides folded in from ../guides.
├── versioned_docs/       # Frozen released versions (v0.7.0)  +  versioned_sidebars/  +  versions.json
├── blog/                 # Posts (.mdx) + authors.yml + tags.yml
├── community/            # index/events (authored) + contribute/code-of-conduct/security/sigs
├── src/
│   ├── pages/            # index.js (placeholder landing), slack.js (Slack redirect)
│   └── css/custom.css    # llm-d theme (brand purple, IBM Plex Sans)
├── static/img/           # Logos, social card, blog avatars, and doc images (img/docs/)
├── scripts/lib/preprocess.mjs   # Build-time Markdown link/image fixups
├── docusaurus.config.js
└── sidebars.js           # Autogenerated; controlled by the docs' _category_.json
```

## How it works

- **Docs** are committed under `docs/` (the dev / "Next" version). Ordering, labels, and
  collapse state come from `_category_.json` files; the sidebar is autogenerated.
- **Versioning** is native: the **v0.7.0** snapshot (the real frozen release) is served at
  `/docs`; the dev docs at `/docs/dev`. The version dropdown is top-right. Cut a new version
  with `npm run version:cut -- 0.8.0`.
- **How-to Guides** are the `llm-d/guides` recipe READMEs rendered as in-site pages; the
  "deploy" links in the well-lit-paths docs route to them (in the same version).
- **Markdown fixups** happen at build time (`scripts/lib/preprocess.mjs`, wired via
  `markdown.preprocessor`) so the source files stay clean: out-of-tree links → GitHub,
  raw-HTML `<img>` → `/img/docs/…`, etc.
- **Community** `contribute` / `code-of-conduct` / `security` / `sigs` pages are generated
  from the repo-root files (`CONTRIBUTING.md`, …) by `node scripts/sync-community.mjs` and
  committed.
